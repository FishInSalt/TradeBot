# TradeBot Phase 1 — 设计文档

## 产品定位

AI 虚拟交易员委托平台。用户通过配置交易偏好，生成个性化的 AI 交易员 Agent，自动分析市场并执行交易。模拟真实世界中委托专业交易员的体验。

## Phase 1 目标

验证核心价值：**AI Agent 能否做出靠谱的交易决策**。

通过 CLI + 模拟盘跑通完整交易链路，用收益指标衡量 AI 决策质量。

## Phase 1 范围

### 包含

- Agent 决策引擎（核心）
- 交易所数据采集（REST + WebSocket）
- 新闻/消息面数据采集
- 技术指标计算
- LLM 集成层（多模型可配置路由）
- 交易执行（OKX 模拟盘，USDT 永续合约）
- CLI 交互界面
- 可配置的人类审批门控
- 核心收益指标展示（CLI）

### 不包含（后续阶段）

- Web UI 界面
- 引导式偏好采集（LLM 对话）
- 多交易会话并行管理
- 金融助手会话
- 多端通知（Telegram 等）
- 宏观经济数据源
- 硬性风控机制（代码层面）
- 实盘交易

## 技术决策

| 项目 | 决策 | 理由 |
|------|------|------|
| 编程语言 | Python + asyncio + type hints + pydantic | AI 生态 + 量化交易生态最强，中频交易场景性能足够 |
| 交易所 | OKX（通过 ccxt 抽象层） | 抽象层支持未来扩展其他交易所 |
| 交易对 | BTC/USDT | 流动性好、数据丰富 |
| 交易类型 | USDT 永续合约 | 支持多空双向，策略空间大 |
| 交易频率 | 中频（15min - 1H 级别） | 匹配 LLM 推理延迟（秒级到半分钟） |
| AI 模型 | 多模型可配置（LLM Router） | 按业务环节分配模型，兼顾效果和成本 |
| 数据库 | SQLite 起步，SQLAlchemy ORM | 零运维，ORM 抽象支持后续迁移 PostgreSQL |
| 数据源 | 技术面 + 消息面 | MVP 够用，终态扩展到全维度 |
| 自主程度 | 全自动/半自动可切换 | 审批门控开关，同一架构支持两种模式 |
| 风控 | MVP 由 AI 软性管控 | 模拟盘无资金风险，不加硬性约束以验证 AI 真实能力 |
| Agent 框架 | Pydantic AI | 轻量不侵入、多模型路由原生支持、async-first、类型安全、已在技术栈中 |

### Agent 核心设计

#### 交易员 Agent（主 Agent）

交易员 Agent 是系统的核心主体，具备四大能力：

**感知能力** — 通过 Tools 感知环境：
- `get_market_data` — 获取 K 线、Ticker 数据
- `get_technical_indicators` — 获取技术指标（MA/RSI/MACD 等）
- `get_news` — 获取新闻和市场情绪
- `get_position` — 查看当前持仓
- `get_account_balance` — 查看账户余额
- `get_trade_history` — 查看交易记录

**规划能力** — LLM 推理 + Chain of Thought：
- 分析当前市场状态
- 评估多空力量对比
- 结合消息面判断方向
- 制定交易计划（入场/出场/仓位/杠杆）
- 决定是否委派子 Agent

**执行能力** — 通过 Tools 执行操作：
- `open_position` — 开仓（多/空、仓位、杠杆）
- `close_position` — 平仓
- `set_stop_loss` — 设置止损
- `set_take_profit` — 设置止盈
- `adjust_leverage` — 调整杠杆
- `delegate_analysis` — 委派子 Agent 执行分析

**记忆能力** — 短期记忆 + 长期记忆：
- 短期记忆：当前分析周期的上下文（对话历史、最近数据）
- 长期记忆：历史交易复盘总结、市场规律认知积累、成功/失败模式记录
- 记忆让 Agent 能从经验中学习，避免重复犯错

#### 交易员人格

用户的交易偏好通过配置转化为 Agent 的 System Prompt，形成独特的"交易员人格"。偏好包括但不限于：
- 风险偏好（激进/稳健/保守）
- 交易风格（趋势跟踪/波段/突破）
- 仓位管理倾向
- 止损/止盈策略偏好

#### 模型能力路由

根据配置的模型能力走不同执行路径：

- **强模型模式**（如 Claude Opus / GPT-4o）：交易员 Agent 独立完成全部分析和决策，信息无损，决策一致性高
- **弱模型模式**（如 Haiku / GPT-4o-mini）：主 Agent 委派子 Agent 分工——市场分析 Agent → 交易决策 Agent → 主 Agent 审核，成本低，各环节专注

通过 Agent 配置中的模型能力等级决定执行策略，不硬编码。

#### 可委派的子 Agent

- **市场分析 Agent**：综合技术面和消息面，输出市场分析报告
- **交易决策 Agent**：基于分析报告，输出具体交易指令
- **复盘总结 Agent**：回顾交易结果，提炼经验，更新长期记忆

### Agent 唤醒机制

模拟真实交易员的"盯盘"行为，两种触发模式：

#### 定时触发

类比交易员每天固定时间看盘。用户可配置分析周期（如每 15min / 1H / 4H），定时唤醒 Agent 做全面市场分析。

实现：asyncio 定时调度器。

#### 条件触发

类比交易员设了价格提醒后去忙别的。Agent 每次分析结束后，通过 `set_alert` tool 自主设定唤醒条件，条件满足时立即唤醒。

条件类型：
- **价格条件**：价格 ≤/≥ 某个值，价格突破某个区间
- **技术指标条件**：RSI 进入超买/超卖区，均线金叉/死叉
- **订单事件**：订单成交、止损/止盈触发
- **新闻事件**：重大新闻关键词匹配、情绪指数剧烈变化

唤醒相关 Tools：
- `set_alert` — 设定唤醒条件
- `cancel_alert` — 取消已设定的唤醒条件
- `list_alerts` — 查看当前所有活跃的唤醒条件

实现：事件监听引擎，持续监听 WebSocket 行情流，匹配 Agent 设定的条件。

### Agent 决策循环

```
⏰ 定时器到期 或 🎯 条件满足
  ↓ 唤醒
👁 感知 → 调用 tools 获取最新行情、新闻、持仓
  ↓
💾 回忆 → 加载相关长期记忆（近期复盘、市场认知）
  ↓
📋 规划 → LLM 推理：分析市场 → 形成观点 → 制定计划
  │       └─ 强模型：自己完成 ｜ 弱模型：委派子 Agent
  ↓
⚡ 执行 → 调用交易 tools 下单（或经审批门控）
  ↓
🎯 设定唤醒条件 → Agent 告诉系统"在什么情况下再叫我"
  ↓
📝 反思 → 记录决策理由，定期复盘更新长期记忆
  ↓
😴 休眠 → 等待下次定时触发或条件触发
  ↻ 循环
```

### 审批门控

通过配置开关控制是否启用人类审批：

- **开启时**：Agent 生成交易指令后暂停执行，通过 CLI 展示交易详情（方向、仓位、杠杆、止损止盈、决策理由），等待用户确认/拒绝
- **关闭时**：Agent 自动执行交易指令，仅在 CLI 中展示日志

### 核心收益指标

CLI 中实时展示以下指标，用于衡量 AI 交易员表现：

- **总收益率** — 累计收益 / 初始资金
- **胜率** — 盈利交易次数 / 总交易次数
- **最大回撤** — 最大峰谷跌幅
- **盈亏比** — 平均盈利金额 / 平均亏损金额
- **交易次数** — 已完成的交易总数
- **当前持仓** — 多头/空头/空仓 + 仓位大小
